# 🎨 Pixel Lab UI 作品社区化改造计划（UI/UX Refactor Plan）

---

## Summary

- 将当前 Pixel Lab UI 从“工具型应用界面”统一升级为“作品分享社区风格界面”，重点提升视觉层级、卡片系统一致性与内容流体验。
- 全局统一圆角、阴影、间距体系，消除后台管理系统风格（直角、重边框、模块化面板感）。
- 首页由“功能模块布局”重构为“内容信息流驱动布局（Feed First）”。
- 强化作品卡片、信息流、用户关系与标签体系的视觉优先级。

---

## Key Changes

---

### 1️⃣ 全局设计语言统一（Global Design System）

#### 修改范围

- 全局 UI 组件样式（按钮 / 卡片 / 面板 / 弹窗）

#### 改动要求

- 移除所有“直角卡片”和强 border 边框风格
- 统一使用：
  - 圆角体系（小卡片 / 大卡片 / Banner 分级圆角）
  - 阴影体系（轻 / 中 / 强三层）
  - 背景层级（背景灰 + 卡片白 + 悬浮层）

#### 视觉目标

- 从「后台系统 UI」→「内容社区 UI（Dribbble风格）」

---

### 2️⃣ 首页结构重构（Home Page Redesign）

#### 当前问题

- 功能模块（工作台 / 手绘板 / 社区）占主导
- 信息呈现偏“工具入口”

#### 改造目标

- 首页变为“作品信息流主入口”

#### 新结构

- Hero区域（轻量）
  - 标题：发现优秀创作
  - 副标题：作品分享与灵感社区
  - CTA：上传作品 / 浏览社区

- 核心区域（主视觉）
  - 作品瀑布流（Feed List）
  - 每张卡片展示作品 + 作者 + 互动数据

- 辅助模块
  - 热门标签（Tag Chips）
  - 推荐创作者（User Cards）

#### 删除/弱化

- 删除“工作台主入口权重”
- 删除“手绘板主展示区域”

---

### 3️⃣ 顶部导航栏（Top Navigation Redesign）

#### 修改范围

- 全局 Header

#### 改动要求

- 保留：
  - Logo
  - 搜索框

- 新增：
  - 上传作品按钮（核心CTA）
  - 消息通知入口
  - 用户头像菜单

- 删除：
  - 工具类入口（工作台 / 手绘板）

#### 目标

- 从“工具导航” → “内容平台导航”

---

### 4️⃣ 底部 Dock 导航（Bottom Dock Redesign）

#### 当前问题

- 视觉偏黑 / 工具感强

#### 改动要求

- 白天模式：
  - 使用浅色玻璃拟态背景
  - 去除纯黑底色

- 夜间模式：
  - 保留深色玻璃质感但降低对比度

- 增加 hover / active 状态微交互
- 图标风格统一线性风格

---

### 5️⃣ 作品卡片系统（Post Card System）

#### 改造目标

统一所有内容展示组件风格

#### 卡片结构统一为：

- 封面图（主视觉）
- 标题（1行）
- 作者信息（头像 + 名称）
- 互动数据：
  - 点赞
  - 评论
  - 收藏

#### 视觉要求

- 圆角卡片
- hover 上浮 + 阴影增强
- 图片轻微缩放动画
- 去掉所有硬边框

---

### 6️⃣ 信息流系统视觉优化（Feed UX）

#### 改动内容

- 信息流采用瀑布流 / Masonry 布局
- 卡片高度自适应
- 图片优先级最高（文本弱化）

#### 视觉层级

- 图片 > 标题 > 作者 > 交互数据

---

### 7️⃣ 个人中心（Profile Page Redesign）

#### 当前问题

- 图片 hover 操作遮罩过重

#### 改造要求

- 移除 hover 遮罩操作层
- 改为：
  - 每张图片右上角固定“三点菜单”

- 菜单内容：
  - 查看
  - 设为公开/私有
  - 删除

#### 页面结构优化

- 用户信息区（头像 / 简介 / 关注）
- 作品瀑布流
- 统计信息弱化展示

---

### 8️⃣ 颜色与视觉体系统一（Color System）

#### 改动要求

- 主色保留绿色（品牌色）
- 背景统一浅灰体系
- 卡片使用纯白或轻灰白
- 禁止大面积纯黑背景（除夜间模式）

---

### 9️⃣ 字体与间距系统（Typography & Spacing）

#### 统一规则

- 标题：16–18px（加粗）
- 正文：13–14px
- 辅助信息：12px灰色

#### 间距

- 模块间距：24–40px
- 卡片内边距：16–20px

---

## Public APIs / Types

- 不修改任何后端接口
- 不新增数据库结构
- 不改变数据字段

---

## Test Plan

### UI验证

- 首页必须以“作品瀑布流”为第一视觉焦点
- 不再存在“工具入口主导视觉”
- 卡片 hover 必须有轻微浮起效果
- 所有卡片圆角统一（无直角元素）

### 功能验证

- 上传作品入口仍可用
- 作品列表正常渲染
- 用户主页作品加载正常
- 三点菜单操作正常

---

## Assumptions

- 当前系统已具备：
  - 上传作品
  - 作品列表
  - 用户系统基础结构

- 本次只做 UI/UX 重构，不改业务逻辑
- 允许使用现有组件但必须统一视觉语言
  可以参考图片中的UI
