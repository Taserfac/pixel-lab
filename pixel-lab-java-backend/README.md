# Pixel Lab Java Servlet Backend

Java Servlet replacement for the original Node/Express backend.

## Stack

- Java 17
- Maven
- Tomcat 9
- Servlet 4.0 (`javax.servlet`)
- JDBC + HikariCP
- MySQL

## Build

```bash
mvn clean package
```

The WAR is built as `target/ROOT.war` so Tomcat exposes APIs at `/api/...`.

## Local Tomcat Run

1. Make sure MySQL has been initialized with the existing `pixel_lab` schema.
2. Configure `pixel-lab-backend/.env` with your local database and DeepSeek values.
3. From the project root, start the backend:

```powershell
cd F:\project\pixel-lab
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\start-java-backend.ps1
```

The script builds the Vue frontend, packages it into `target/ROOT.war`, deploys it to local Tomcat, loads `pixel-lab-backend/.env`, and waits for `http://127.0.0.1:8080/api/health`.

Open the site directly at:

```text
http://localhost:8080
```

Frontend routes such as `http://localhost:8080/dashboard` and `http://localhost:8080/draw` are served by Tomcat too, so you do not need to run `npm run dev` for normal local use.

Stop the local backend with:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\stop-java-backend.ps1
```

The API base URL is `http://localhost:8080`.

## Runtime Config

Default config lives in `src/main/resources/app.properties`.
Every property can be overridden by an environment variable:

- `DB_HOST`
- `DB_PORT`
- `DB_NAME`
- `DB_USER`
- `DB_PASSWORD`
- `CORS_ALLOWED_ORIGINS`
- `UPLOAD_PATH`
- `DEEPSEEK_API_KEY`
- `DEEPSEEK_BASE_URL`
- `DEEPSEEK_CHAT_MODEL`
- `DEEPSEEK_IMAGE_MODEL`
- `COMFYUI_BASE_URL`

Windows examples:

```powershell
setx DB_HOST localhost
setx DB_PORT 3306
setx DB_NAME pixel_lab
setx DB_USER root
setx DB_PASSWORD "your_mysql_password"
setx DEEPSEEK_API_KEY "your_deepseek_key"
```

Restart Tomcat after changing environment variables.

## Session Login

Login creates an `HttpSession` and stores:

- `LOGIN_USER`
- `userId`
- `role`

The browser keeps login state through the `JSESSIONID` cookie. Frontend requests must use credentials/cookies.

## Course Acceptance Points

- Session tracking: `HttpSession + JSESSIONID`.
- Servlet filters: encoding, CORS, request log, auth, admin auth.
- Servlet listeners: application startup/shutdown, session lifecycle, request lifecycle.

## Nginx Production Proxy

For Alibaba Cloud deployment, use the ready-to-upload package generated by:

```powershell
cd F:\project\pixel-lab
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\deploy\package-release.ps1
```

Then follow `deploy/README.md`.

Serve the Vue `dist` directory with Nginx and proxy backend paths to Tomcat:

```nginx
location /api/ {
  proxy_pass http://127.0.0.1:8080/api/;
  proxy_set_header Host $host;
  proxy_set_header X-Forwarded-Proto $scheme;
  proxy_set_header X-Forwarded-Host $host;
}

location /uploads/ {
  proxy_pass http://127.0.0.1:8080/uploads/;
  proxy_set_header Host $host;
  proxy_set_header X-Forwarded-Proto $scheme;
  proxy_set_header X-Forwarded-Host $host;
}
```
